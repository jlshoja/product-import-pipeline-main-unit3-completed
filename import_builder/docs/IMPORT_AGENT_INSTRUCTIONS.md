## Overview

## File Purpose Definitions

### 1. Full Catalog (`purpose: "full_catalog"`)
**File Pattern**: `woocommerce_import_*.xlsx`
**Update Scope**: `all_fields`
**Import Priority**: 1

**Behavior**:
- Complete catalog replacement
- **Create** new products (SKUs not in WooCommerce)
- **Update** ALL fields for existing products (title, price, description, attributes, images, stock, categories, SEO)
- **Delete** products in WooCommerce but NOT in this file (if full-sync mode enabled)
- Images: Link all images from `input/images/` by SKU prefix match

**When Used**: First run, manual full refresh, major site redesign

---

### 2. New Products (`purpose: "new_products"`)
**File Pattern**: `woocommerce_new_*.xlsx`
**Update Scope**: `create_only`
**Import Priority**: 2

**Behavior**:
- **CREATE ONLY** - never update existing products
- For each SKU in file:
  - If SKU exists in WooCommerce → **SKIP** (log warning)
  - If SKU does not exist → Create product with all fields from file
- Images: Link images for these SKUs only

**When Used**: Incremental run with new products detected

---

### 3. Updated Products (`purpose: "updated_products"`)
**File Pattern**: `woocommerce_update_*.xlsx`
**Update Scope**: `price|color_variants|stock`
**Import Priority**: 3

**Behavior**:
- **UPDATE ONLY** - never create new products
- For each SKU in file:
  - If SKU exists in WooCommerce → Update only these fields:
    - `regular_price`, `sale_price`
    - Color attribute variations (add new color variants)
    - Stock quantity/status
    - Variation images for new colors
  - If SKU does not exist → **SKIP** (log warning)
- Do NOT modify: title, description, categories, other attributes

**When Used**: Incremental run with price changes or new color variants

## Execution Priority Order

The `import_priority` field determines execution order:

| Priority | Purpose | When to Run |
|----------|---------|-------------|
| 1 | `full_catalog` | Alone (mutually exclusive with incremental) |
| 2 | `new_products` | Before updates (creates parent products for variations) |
| 3 | `updated_products` | After new products |

**Critical Rule**: 
- If `full_catalog` file exists → Run ONLY that file (ignore any incremental files)
- If NO `full_catalog` → Run `new_products` (priority 2) then `updated_products` (priority 3)

## Image Handling

### Source
All images are pre-copied to `input/images/` with naming:
```
{SKU}{letter}_{color}.webp
```
Examples:
- `9266a_black.webp` (main image for SKU 9266, color black)
- `9266b_blue.webp` (gallery image)
- `9266c.webp` (gallery image, no color)

### Matching Rules
1. **Parent product**: Match by SKU prefix (numeric part before letter)
   - SKU `9266` → images `9266a_*.webp`, `9266b_*.webp`, etc.
2. **Variation images**: Color suffix matches variation color attribute
   - Variation `color: black` → image `9266a_black.webp`
3. **Main image**: First alphabetical (`a`) image becomes product gallery first image
4. **Overwrite mode**: Replace existing images with same filename

### Missing Images
- Log warning: `Missing images for SKU XXXX: expected 3, found 1`
- Continue import without failing

## Decision Tree (Pseudocode)

```python
manifest = read_manifest()
files = manifest['files']

full_catalog = [f for f in files if f['purpose'] == 'full_catalog']
new_products = [f for f in files if f['purpose'] == 'new_products']
updated_products = [f for f in files if f['purpose'] == 'updated_products']

if full_catalog:
    # FULL MODE - mutually exclusive
    if new_products or updated_products:
        alert("WARNING: Full catalog + incremental files present. Running full only.")
    run_full_import(full_catalog[0])
else:
    # INCREMENTAL MODE
    if new_products:
        run_new_products_import(new_products[0])
    if updated_products:
        run_updated_products_import(updated_products[0])
    if not new_products and not updated_products:
        alert("No import files found in manifest")
```

## Import Action Functions

### `run_full_import(file_info)`
1. Read XLSX file
2. For each row:
   - Determine if product exists (by SKU)
   - If new → `wc_create_product(row)`
   - If exists → `wc_update_product(row, fields='all')`
3. If full-sync enabled → `wc_delete_missing_products(imported_skus)`
4. Link images for all imported SKUs

### `run_new_products_import(file_info)`
1. Read XLSX file
2. For each row:
   - If SKU exists in WC → log skip, continue
   - Else → `wc_create_product(row)`
3. Link images for created SKUs only

### `run_updated_products_import(file_info)`
1. Read XLSX file
2. For each row:
   - If SKU not in WC → log skip, continue
   - Else → `wc_update_product(row, fields=['price', 'color_variants', 'stock'])`
3. Link variation images for new color variants

## Error Handling

| Scenario | Action |
|----------|--------|
| Manifest missing | Alert human, do not proceed |
| Manifest invalid JSON | Alert human, do not proceed |
| Referenced XLSX missing | Alert human, do not proceed |
| SKU mismatch (file has SKU not in WC for update) | Log warning, skip row |
| SKU conflict (file has SKU in WC for new) | Log warning, skip row |
| Image copy failed | Log error, continue |
| WC API error | Retry 3x, then alert human |

---

## Logging Requirements

Log every:
- File read (path, row count)
- Product created (SKU, type)
- Product updated (SKU, fields changed)
- Product skipped (SKU, reason)
- Image linked (SKU, image filename)
- Image missing (SKU, expected vs found)
- Errors with full context

---

## Manifest Copy Log Verification

The manifest's `copy_log` array contains all file copy operations. Verify:
- All Excel files: `success: true`
- Images: `success: true` for expected count
- Any `success: false` → alert human before import

### What the Import Agent Uses at Runtime

**Only 2 things needed from this project:**
1. **The Manifest JSON** (`input/import_manifest_*.json`) - contains everything: which XLSX files exist, each file's `purpose`, `update_scope`, `import_priority`, image folder info, copy verification log
2. **The XLSX files + images** in the input folders (auto-copied by this project)

### No Manual Copy Needed

This project **automatically copies** all files to the Import Project folders after every successful run via:
- `import_builder/import_copier.py` → copies XLSX + images (overwrite mode)
- `import_builder/manifest_generator.py` → creates manifest in import folder

The Import Project just needs to **watch the `input/` folder** for new manifest files and process them.
